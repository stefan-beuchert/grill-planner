```

                               all four tabs (rendered up front, Tabs

  admin/page.tsx               Admin login + cross-party dashboard
  error.tsx / global-error.tsx Error boundaries

components/
  party/     One component per tab section, e.g. shopping-list-section.tsx,
```

# Naming conventions

- Components: PascalCase function in a kebab-case file
  (`contribution-list.tsx` → `ContributionList`).
- Server Actions: verb-first camelCase (`setContribution`, `addItem`,
  `joinParty`, `adminRemoveGuest`).
- Validation schema factories: `xSchema` (`itemNameSchema`,
  `partyFormSchema`); some take `t: Dictionary` for localized error
  messages, some don't (`quantitySchema`, `rideStatusSchema`).
- Inferred form types: `XValues` (`ItemNameValues`, `PartyFormValues`).

# Central docs, and what each one is actually for

- **CLAUDE.md** — stack, coding conventions, dev workflow, the agent
  pipeline itself.
- **PRODUCT.md** — product rules and behavior (the Contribution Ledger
  rules, Admin Mode & Party Organizers, tab layout, design principles).
- **ARCHITECTURE.md** — technical reference: data model (generated section),
  auth mechanisms, external services, a worked request-flow example, error
  handling, dev environment, testing.
- **README.md** — setup and day-to-day commands.
